NOTAS TÉCNICAS
------------------------------------------------------------

- Para ver el sitio con todas las funciones activas (carga de JSON),
  se necesita un servidor local. Opciones:
    a) Extensión "Live Server" de VS Code (recomendado)
    b) Ejecutar en terminal: python -m http.server 8765
       Luego abrir: http://localhost:8765

- El sitio usa particles.js desde CDN. Si no hay internet,
  las partículas no cargarán pero el resto del sitio sí funciona.

- Archivos principales a NO modificar sin conocimiento CSS/JS:
    css/style.css  (solo si se quiere cambiar colores/fuentes)
    js/main.js     (solo si se quiere agregar funcionalidad)

- Archivos SEGUROS de editar libremente:
    data/noticias.json

    data/capitulos.json
    index.html  (secciones de texto y links)

────────────────────────────────────────────────────────────
 5. ARQUITECTURA Y DECISIONES TÉCNICAS
────────────────────────────────────────────────────────────

  ▸ Sin build system: archivos estáticos puros (HTML/CSS/JS).
    Razón: facilita edición por parte del equipo no-técnico
    y deploy en cualquier hosting (GitHub Pages, Netlify, etc.).

  ▸ Navbar y footer hardcodeados en cada página.
    Trade-off: cambios estructurales requieren editar 6 archivos.
    Si esto se vuelve carga, migrar a Eleventy/Astro/Hugo.

  ▸ Detección de página por body[data-page="..."]:
      home → scrollspy activo
      capitulos / noticias / concursos / galeria / contacto →
        link activo marcado en HTML, scrollspy desactivado

  ▸ Detección de modo por contenedor[data-mode="..."]:
      preview → home, listas limitadas, sin filtros
      full    → subpáginas, listas completas, filtros activos

  ▸ JSON como fuente única de contenido editable.
    El equipo puede actualizar capítulos/noticias/concursos sin
    tocar código (HTML/CSS/JS).

  ▸ Formulario de contacto: detecta automáticamente si el atributo
    action contiene "formspree.io" y cambia a envío real con fetch.
    Mientras tanto, modo demo muestra confirmación visual.

────────────────────────────────────────────────────────────
 6. CÓMO EJECUTAR EL SITIO
────────────────────────────────────────────────────────────

Requiere servidor local porque las páginas usan fetch() para
cargar los JSON (no funcionan abriendo el archivo directamente
en el navegador, file://).

Opción A – Python (recomendado, ya disponible en Windows):
    cd C:\Users\PC\Desktop\IEEE_Rama_General_Web
    python -m http.server 8765
    Abrir: http://localhost:8765

Opción B – VS Code Live Server:
    Instalar extensión "Live Server" → clic derecho en index.html
    → "Open with Live Server"

Opción C – Node.js:
    npx serve .

Para detener el servidor: Ctrl+C en la terminal.
